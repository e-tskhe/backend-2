Залила файлы в каталоге files на веб-сервер через GitHub и команды git clone в PuTTY. Проверила работоспособность index.php

Подключилась к серверу с помощью PuTTY, используя протокол telnet (это сетевой протокол, работающий по TCP-соединению, предназначенный для ручного управления удаленными машинами с помощью команд консольного терминала), 80 порт (стандартный канал для связи по протоколу HTTP для обмена данными между веб-серверами и клиентами) и тип соединения Raw (режим соединения в PuTTY, который позволяет подключаться к любым текстовым протоколам, работающим по системе «запрос — ответ»).

Запросы записала в текстовый файл, и далее через буфер обмена мышью копировала их из файла.

1. Получить главную страницу методом GET в протоколе HTTP 1.0

Сервер возвращает содержимое с заголовком Apache2 Ubuntu Default Page: It works. Это страница по умолчанию, используемая для тестирования правильной работы сервера Apache2 после установки в системах Ubuntu

Date - дата обработки запроса
Server - версия сервера, который обрабатывает запрос
Last-Modified - дата последнего изменения страницы
ETag - закрытый идентификатор, присвоенный веб-сервером определённой версии ресурса
Accept-Ranges: bytes - указывает, что можно запрашивать данные с сервера фрагментами, указывая смещение в байтах
Content-Length - длина содержимого в байтах
Vary: Accept-Encoding - указывает, что сервер может отдавать разные версии контента (сжатые или несжатые) в зависимости от заголовка Accept-Encoding в запросе клиента
Connection: close - соединение будет закрыто после передачи ответа
Content-Type - указывает на тип содержимого
